## 推荐默认权重

| 因素 | 权重 |
| --- | ---: |
| 兼容性与场景匹配 | 35% |
| 历史质量 | 25% |
| 交付表现 | 20% |
| 价格合理性 | 15% |
| 白名单加成 | 5% |

硬约束失败或命中有效黑名单时不进入评分。权重仅为默认配置，任何变更必须生成新策略版本并保存到推荐快照。

## 变更流程

1. 建立独立 Issue，说明业务原因、影响需求编号和迁移/兼容风险。
2. 更新本文件、需求、架构、数据库和接口中的受影响部分。
3. 更新验收场景、自动化测试和历史数据处理策略。
4. 另一名开发者批准后实施；高风险决策由业务负责人确认。
