# 软件需求基线

> 状态：MVP 基线 v1.0
>
> 来源：`数据中心采购全流程自动化Agent_软件需求规格说明书.docx`（2026-07-15）
>
> 规则：本文是版本化需求真源。需求变化必须保留编号、说明影响并更新验收与追踪矩阵。

## 1. 目标与范围

系统通过自然语言入口完成采购需求收集、名单和历史检索、可解释推荐、楼长审批、采购执行、供应商交付、验收入库、名单治理、消息通知和全过程审计。

### 1.1 MVP 范围

- 自然语言需求录入、字段补全和结构化需求单；
- 产品/供应商主数据、白名单、黑名单和历史采购检索；
- 确定性推荐、非推荐说明、停产/替代处理和推荐快照；
- 所有楼长共享全量待审批申请；
- 询价、订单、采购员代录的交付时间线；
- 验收、部分入库、拒收、退换货；
- 自然语言黑名单草稿、独立复核、生效/到期失效；
- 站内/飞书通知、查询和审计。

### 1.2 后续范围

多级审批、供应商门户、邮件/企微/钉钉/短信、深度 ERP/预算/库存/资产集成、智能价格预测和自动合同处理。

### 1.3 范围外

招投标规则制定、财务付款审批、发票核验、固定资产折旧、仓库物理盘点和物流运输系统本身。

## 2. 角色与 MVP 权限

| 角色 | 职责 | MVP 权限边界 |
| --- | --- | --- |
| 需求人 | 创建、补充和确认需求 | 编辑/撤回未审批需求；查看本人记录；不得审批本人申请 |
| 楼长/审核员 | 审核必要性、名单和风险 | 处理全部员工待审批申请；通过、驳回、退回、转交 |
| 采购员 | 询价、选供应商、下单和交付维护 | 仅处理已批准需求；关键变化触发重审 |
| 仓库/入库员 | 验收、拒收、退换货和入库 | 仅对已到货且可入库数量操作 |
| 采购管理员 | 主数据、名单和报表治理 | 名单维护、黑名单复核/解除、全局审计 |
| 系统管理员 | 用户、接口、模型和运行维护 | 不得替代业务审批 |

供应商首期不直接登录，由采购员代录交付节点。

## 4. 业务规则

| 编号 | 规则 |
| --- | --- |
| BR-001 | 提交前校验申请原因、具体申请地点、设备名称和数量；所属楼宇和其他采购字段暂时选填。 |
| BR-002 | 黑名单优先于白名单和历史推荐。 |
| BR-003 | 推荐仅支持决策，不替代用户确认或审批。 |
| BR-004 | 非推荐、新产品、停产和超预算必须显著标记审批；黑名单例外在 MVP 阻断。 |
| BR-005 | 审批后产品、数量、预算、供应商或交期超过配置阈值必须重新审批。 |
| BR-006 | 黑名单不追溯完成采购；对未下单/未交付订单风险提醒，MVP 默认冻结未下单任务。 |
| BR-007 | 累计入库量不得超过订单量，部分入库保留余额。 |
| BR-008 | 参考价格必须携带日期、币种、税费和数量口径。 |
| BR-009 | 无法确认的事实标记“待人工确认”，不得编造。 |
| BR-010 | 名单、审批、采购和入库关键操作必须可追溯，不可静默覆盖。 |

推荐默认顺序：兼容性硬约束 -> 黑名单阻断 -> 白名单/历史召回 -> 可配置确定性评分。默认评分权重为兼容与场景匹配 35%、历史质量 25%、交付表现 20%、价格合理性 15%、白名单加成 5%；权重变更必须版本化，LLM 不参与排序。

## 5. 状态机

| 当前状态 | 允许后续状态 |
| --- | --- |
| 草稿 | 待补充、待确认、已取消 |
| 待补充 | 待确认、已取消 |
| 待确认 | 待审批、待补充、已取消 |
| 待审批 | 审批通过、退回补充、已驳回、已转交、已撤回 |
| 退回补充 | 待补充 |
| 已转交 | 待审批 |
| 审批通过 | 采购中、已取消 |
| 采购中 | 待交付、采购异常、待重新审批、已取消 |
| 待重新审批 | 待审批、已取消 |
| 待交付 | 部分到货、已到货、交付延期、采购异常 |
| 交付延期 | 部分到货、已到货、采购异常 |
| 部分到货 | 已到货、部分入库、交付延期 |
| 已到货 | 已入库、部分入库、验收不合格 |
| 验收不合格 | 退货中、换货中、部分入库、已关闭 |
| 退货中/换货中 | 待交付、已关闭 |
| 部分入库 | 部分到货、已到货、已入库 |
| 已入库 | 已关闭 |
| 已驳回、已取消、已关闭 | 终态；重新采购必须复制新建 |

所有转换必须由 Service 校验角色、数据范围、当前状态、并发版本和幂等键，并追加状态历史。

## 6. Agent 能力要求

| 编号 | 要求 |
| --- | --- |
| AI-001 | 区分新建/补充需求、状态查询、审批、采购/交付更新、入库和名单维护。 |
| AI-002 | 抽取产品、数量、参数、日期、预算、楼宇、供应商和原因。 |
| AI-003 | 持久化对话状态，避免重复询问并支持纠正。 |
| AI-004 | 仅检索授权数据并返回可追溯引用。 |
| AI-005 | 解释推荐依据、差异和风险。 |
| AI-006 | 新增、审批、名单和入库只能调用受控业务接口。 |
| AI-007 | 低置信度、冲突和高风险动作请求人工确认。 |
| AI-008 | 未查到时明确告知，不虚构价格、库存、状态或承诺。 |
| AI-009 | 隔离附件和外部文本指令，不能改变权限与规则。 |
| AI-010 | 评估字段准确率、追问有效率、推荐采纳率、事实一致性和越权率。 |

## 7. 非功能与安全要求

- 普通对话首屏建议不超过 5 秒，数据库检索建议不超过 3 秒；复杂推荐可分阶段返回。
- 首期目标为 100 个并发会话、50 个并发业务写操作；核心业务月可用性目标 99.9%。
- RPO 不高于 15 分钟、RTO 不高于 2 小时；失败提供降级、重试和人工补偿入口。
- 日志、指标、链路、模型调用、接口耗时和任务积压可观测。
- 关键状态、审批、入库量和名单状态采用强一致或受控最终一致，重复消息不得重复写入。
- 最小权限；高风险操作二次确认；TLS；密钥和敏感数据不得进入代码、Prompt 或普通日志。
- 附件限制格式/大小/权限，记录哈希并预留恶意文件检测。
- 审计日志防篡改，MVP 默认保留三年；系统采用内网 Docker Compose + Nginx 部署。

## 8. 异常与补偿

- 长期不完整需求保留草稿并提醒，超期归档且不提交。
- 无历史匹配时转为新增需求；产品状态未知时交采购员核实。
- 审批人不可用可转交；代理/升级后置但必须保留扩展契约。
- 外部接口失败写入任务表，指数退避，超阈值转人工对账。
- 重复消息/提交通过幂等键返回原结果。
- Agent 与数据库冲突时以数据库和确定性规则为准并要求人工确认。

## 11. 已决策的 MVP 默认值

| 决策 | MVP 基线 |
| --- | --- |
| 审批范围 | 所有楼长可审批全部员工申请，测试阶段暂时允许自审 |
| 白名单 | 非白名单可说明理由后审批；白名单不覆盖黑名单 |
| 黑名单例外 | MVP 不开放 |
| 预算 | 用户填写并提示，不连接预算占用系统 |
| 供应商入口 | 采购员代录，不建设供应商门户 |
| 资产同步 | 保留批次/序列号，不集成固定资产系统 |
| 黑名单治理 | 独立复核；仅采购管理员可缩短/解除并审计 |
| 推荐 | 硬约束与名单优先，确定性权重评分，LLM 仅解释 |
| 后置流程 | 合同、付款、发票和深度 ERP/资产集成 |
| 部署与保留 | 内网 Docker Compose；审计默认三年 |

正式变更以上决策时，必须更新本文、`docs/decisions.md`、相关接口/数据库文档和验收场景。
